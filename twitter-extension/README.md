# Twitter Comment Filter

一个可直接加载到浏览器的 `Manifest V3` 插件，用于在 `x.com` / `twitter.com` 推文详情页按自定义文本或正则规则移除评论。

## 功能

- 自定义添加多条屏蔽文本
- 支持普通文本规则和正则表达式规则
- 普通文本会忽略大小写、空格、常见标点和链接
- 支持规则导出与导入（可跨设备迁移）
- 支持通过 GitHub raw URL 同步云端规则
- 本地规则和云端规则会同时参与匹配
- 自动监听评论区动态加载内容

## 使用方式

1. 打开 Chrome 或 Edge 的扩展管理页。
2. 开启“开发者模式”。
3. 选择“加载已解压的扩展程序”。
4. 选中当前目录：`twitter-extension`
5. 打开任意 `x.com/.../status/...` 或 `twitter.com/.../status/...` 页面。
6. 点击插件图标，添加你要屏蔽的文本或正则规则。

## 云端规则

1. 将导出的规则 JSON 文件上传到 GitHub 仓库。
2. 在 GitHub 上获取该文件的 raw URL（`https://raw.githubusercontent.com/...`）。
3. 将 raw URL 填入插件的"GitHub Raw URL"输入框，点击"同步云端规则"。

规则文件所在仓库应为公开仓库，或 raw URL 必须能在浏览器里无鉴权访问。

规则 JSON 格式：

```json
{
  "exportedAt": "2026-05-14T00:00:00.000Z",
  "patterns": ["广告", "/推广.{0,4}链接/i"]
}
```

也支持纯文本格式，每行一条规则。

## 匹配说明

- 普通文本规则：去掉空格、标点、链接后做直接包含匹配
- 正则规则：使用 `/pattern/flags` 格式，按评论原文匹配
- 当前实现默认只在推文详情页处理，避免误伤首页时间线
